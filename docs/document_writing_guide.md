# Document Writing Guide

## 서술 목적

이 문서는 `docs/` 내부 문서를 일관된 레이아웃과 서술체로 작성하기 위한 기준 문서다. 목적은 문서 간 중복을 줄이고, AI가 새 문서를 쓰거나 기존 문서를 수정할 때 같은 형식과 같은 문체를 유지하게 만드는 데 있다.

## 다루는 범위

- `docs/` 내부의 `workflow/`, `research/`, `references/` 문서
- 문서의 공통 레이아웃
- 문서 타입별 권장 구조
- AI가 따라야 할 문체와 서술 규칙

## 다루지 않는 범위

- 코드 주석이나 커밋 메시지 규칙
- 채팅 응답의 말투
- 논문 본문 전체의 포맷팅 규칙

## 상세 서술

### 1. 공통 레이아웃

`docs/` 내부 문서는 아래 순서를 기본으로 둔다.

1. `문서 제목`
2. `서술 목적`
3. `다루는 범위`
4. `다루지 않는 범위`
5. `상세 서술`
6. `문서 정리`
7. `관련 문서`

모든 문서가 같은 길이일 필요는 없다. 다만 문서 첫부분에서 `무엇을 다루는지`, `왜 필요한지`, `무엇을 여기서 다루지 않는지`는 반드시 드러나야 한다.

### 2. 문서 타입별 권장 구조

`Workflow` 문서는 실무 흐름과 현재 상태를 정리한다. 기본 구조는 `목적 -> 현재 기준 -> 현재 상태 -> 다음 작업 -> 관련 문서`가 적절하다.

`Spec` 또는 `Protocol` 문서는 규칙과 스키마를 고정한다. 기본 구조는 `목적 -> 적용 범위 -> 규칙 -> 스키마/키 -> 예외 -> 문서 정리 -> 관련 문서`를 권장한다.

`Guide` 문서는 개념 설명과 이해를 돕는다. 기본 구조는 `목적 -> 왜 필요한가 -> 핵심 개념 -> 예시/오해 -> 문서 정리 -> 관련 문서`가 적절하다.

`Review` 문서는 문헌이나 배경지식을 압축 정리한다. 기본 구조는 `목적 -> quick map -> 축별 요약 -> 현재 프로젝트에 주는 결론 -> 관련 문서`를 권장한다.

### 3. AI 서술체 규칙

문서 본문은 기본적으로 `-다`체를 쓴다. 채팅 답변의 존댓말과 문서 본문을 섞지 않는다.

첫 문장은 결론형으로 쓴다. 예를 들면 `이 문서는 X를 고정한다`, `이 규칙의 목적은 Y다`, `핵심은 Z다`처럼 시작하는 편이 좋다.

한 문단에는 한 주장만 둔다. 정의, 이유, 예외, 해석을 한 문단에 모두 넣지 않는다.

문단은 짧게 유지한다. 서론은 보통 2~4문장, 일반 문단은 2~4문장을 권장한다.

같은 개념은 한 문서에서 한 번만 길게 정의한다. 다른 문서에서 이미 정의한 개념은 짧게만 언급하고 `관련 문서`로 넘긴다.

### 4. 표현 규칙

technical term는 영어를 유지하되, 설명은 한국어로 쓴다. 예를 들면 `pinball loss`, `quantile crossing`, `event table`, `holdout`은 영어 그대로 두고 역할과 이유는 한국어로 설명한다.

과도한 수사적 표현은 피한다. `정말`, `매우 중요하다`, `당연히` 같은 표현보다 구체적인 이유를 적는다.

리스트와 표는 구조가 필요한 경우에만 쓴다. 규칙, 비교, 스키마, 체크리스트처럼 구조가 분명한 내용이 아니면 짧은 산문을 우선한다.

### 5. 중복 방지 규칙

문서마다 `주 질문`은 하나만 둔다. 한 문서가 여러 질문에 답하기 시작하면 다른 문서와 쉽게 겹친다.

`왜 중요한가`는 같은 문서에서 반복하지 않는다. 문서 초반이나 해당 개념의 첫 등장 시점에서만 설명한다.

이미 다른 문서에서 자세히 다룬 내용은 다시 길게 풀지 않는다. 현재 문서에서 필요한 최소 설명만 남기고 `관련 문서`로 연결한다.

### 6. 관련 문서 섹션 규칙

`관련 문서`는 읽는 순서를 강제하는 칸이 아니다. 이 문서와 직접 맞닿는 문서, 규칙을 이어받는 문서, 보완 설명 문서를 적는 칸으로 쓴다.

보통 2~5개 문서면 충분하다. 여기에는 새로운 설명을 추가하지 않고, 연결 이유를 짧게만 적는다.

## 문서 정리

앞으로 `docs/` 내부 문서는 공통 레이아웃과 `-다`체를 기본으로 삼는다. AI는 `결론 우선`, `한 문단 한 주장`, `중복 설명 대신 관련 문서 연결`을 우선 규칙으로 따른다.
